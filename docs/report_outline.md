# Черновик структуры отчёта по курсовой работе

## 1. Титульный лист (шаблон)
- Министерство/вуз/кафедра
- Дисциплина
- Тема: «Разработка учебного мини-компилятора с использованием Flex, Bison и LLVM»
- Вариант: архитектура RISC-V, спецконструкция — тернарный оператор, Go-style синтаксис
- Студент: ФИО, группа
- Руководитель: ФИО, должность
- Город, год

## 2. Задание
В рамках курсовой требовалось реализовать мини-компилятор языка программирования с поддержкой базовых конструкций управления и функций.  
Обязательные технологии: Flex, GNU Bison, C++17, LLVM.  
Целевой backend: генерация объектного файла под RISC-V и демонстрация интеграции с внешним `main.c`.

Ключевые требования:
- отдельные фазы lexer/parser/sema/codegen;
- построение AST на этапе parser;
- отдельный semantic analyzer;
- генерация LLVM IR и объектного файла;
- наличие тестов и документации.

## 3. Лексическая спецификация
Лексический анализатор реализован на Flex и распознаёт:
- ключевые слова: `func`, `var`, `if`, `else`, `for`, `return`, типы;
- идентификаторы;
- литералы: целые, вещественные, булевы;
- операторы: арифметические, логические, сравнения, `?:`;
- разделители: `(){} , ; :`.

Поддержаны однострочные (`//`) и многострочные (`/* ... */`) комментарии.  
Для всех токенов ведётся учёт позиции (`line`, `column`) для диагностики ошибок.

## 4. Грамматика
Грамматика реализована в GNU Bison (LALR(1)).  
В грамматике заданы приоритеты и ассоциативность операторов, включая тернарный оператор.

Основные синтаксические конструкции:
- объявление функций;
- блочные statements;
- объявления переменных и присваивания;
- `if-else`, `for` (две формы);
- выражения с вызовами функций, унарными/бинарными и тернарным оператором.

## 5. Лексер и парсер
Парсер строит AST, а не LLVM IR, что обеспечивает корректное разделение фаз компиляции.  
Каждый узел AST содержит информацию о позиции в исходном файле.  
При синтаксических ошибках формируется сообщение в едином формате:

`<file>:<line>:<col>: error: <message>`.

## 6. Семантические правила
В языке используются типы `int`, `float`, `bool`, `void`.  
Политика преобразований строгая: разрешено только неявное `int -> float`.

Проверяются правила:
- объявления и области видимости;
- корректность операций по типам;
- условия `if`/`for` только `bool`;
- корректность return;
- корректность вызовов функций;
- совместимость ветвей тернарного оператора.

## 7. Семантический анализатор
Семантический анализатор реализован отдельным модулем:
- глобальная таблица функций;
- стек таблиц символов для переменных (scope stack).

Сначала регистрируются сигнатуры функций, затем проверяются тела функций.  
При ошибках sema кодогенерация не выполняется.

## 8. Преобразование AST -> IR -> obj
После успешной sema AST транслируется в LLVM IR:
- типы языка отображаются в LLVM-типы;
- локальные переменные реализованы через `alloca` + `load/store`;
- ветвления и циклы через basic blocks.

Затем модуль верифицируется и может быть:
- сохранён как `.ll`;
- скомпилирован в `.o` под `riscv64-unknown-linux-gnu`.

## 9. Кодогенератор
Кодогенератор реализует:
- арифметику, сравнения, логические выражения;
- short-circuit для `&&` и `||`;
- тернарный оператор `?:` через ветвление и `PHI`;
- `if/else`, `for`, `return`.

Используются `verifyFunction` и `verifyModule` для проверки корректности генерируемого IR.

## 10. Тесты
В проекте выделены:
- позитивные тесты;
- негативные синтаксические тесты;
- негативные семантические тесты;
- интеграционный тест.

Тестовый скрипт автоматически проверяет ожидаемый успех/ошибку компиляции и наличие диагностик.

## 11. Использование компилятора
Последовательность использования:
1. Сборка проекта через CMake.
2. Запуск `lorsc` на исходнике с параметром `-o`.
3. (Опционально) генерация `.ll` через `--emit-llvm`.
4. Линковка полученного `.o` с `runtime/main.c`.
5. Запуск RISC-V бинарника через `qemu-riscv64`.

## 12. Результаты тестирования
В итоговой версии:
- позитивные тесты подтверждают поддержку всех обязательных конструкций;
- негативные тесты подтверждают работу диагностик parser/sema;
- интеграционный сценарий подтверждает генерацию объектного файла для RISC-V и возможность запуска через эмулятор.

Для финальной версии отчёта в этот раздел добавляются:
- таблица «тест -> ожидаемый результат -> фактический результат»;
- скриншоты/логи команд сборки, компиляции и запуска.
